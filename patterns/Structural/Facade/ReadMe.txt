Простой паттерн используеться в случае если у нас есть некая сложная логика которая может проходить "под капотом".
Создаеться класс в котором делаються все нужные операции а пользователю выделяються только нужные методы для выполнения этих операциый(походе не API)
По сути мы прячем низкоуровненые операции, замещая только высокоуровневыми, но прижелании можно использовать найтивные методы
